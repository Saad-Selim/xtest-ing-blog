## What is Automated Smoke Testing?

Smoke testing, also known as build verification testing, is a type of software testing that consists of a non-exhaustive set of tests aimed at ensuring that the most critical functions of a software system are working as expected. The term 'smoke testing' originally comes from hardware testing, where the device passed the test if it did not emit smoke when first switched on. In software testing, smoke tests serve a similar purpose; they are the initial tests run to check the basic functionality of the software before proceeding to more rigorous testing.

Automated smoke testing takes this to a whole new level. With automation, smoke tests can be run frequently and consistently, without manual intervention, saving both time and resources.
